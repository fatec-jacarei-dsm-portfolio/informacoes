Portfólio Digital

Orientações Oficiais - Desenvolvimento de Software Multiplataforma

Objetivo

O Portfólio Digital tem como objetivo apresentar a trajetória acadêmica, profissional e pessoal do aluno ao longo do curso de Desenvolvimento de Software Multiplataforma da Fatec Jacareí.

1. Repositório e Publicação

Cada aluno possui um repositório institucional no formato:

https://github.com/fatec-jacarei-dsm-portfolio/ra[RA_DO_ALUNO]

Todo o conteúdo do portfólio deve ser mantido dentro da pasta:

/docs

O portfólio é publicado pela instituição através do GitHub Pages:

https://fatec-jacarei-dsm-portfolio.github.io/ra[RA_DO_ALUNO]/

O link informado no README deve apontar obrigatoriamente para esse endereço.

Não serão considerados links para outras plataformas de hospedagem.

Pré-requisito

O professor responsável criará o repositório de cada aluno. O aluno deverá aceitar o convite pelo GitHub para realizar os envios.

Termo de Consentimento

Todo aluno deverá assinar um termo declarando ciência de que a Fatec armazenará seus projetos e vídeos para fins de auditoria acadêmica.

2. README

O arquivo README.md deve permanecer na raiz do repositório.

Requisitos do README

  • Utilizar o template fornecido.
  • Informar corretamente a turma de ingresso.
  • Informar o link do portfólio online.
  • Manter atualizados os links dos vídeos.
  • Preservar os vídeos produzidos nas avaliações anteriores.

3. Estrutura do Portfólio

Apresentação

  • Foto do aluno.
  • Breve apresentação pessoal.
  • Objetivos profissionais ou áreas de interesse.

Projetos

Os projetos devem ser organizados em:

  • Acadêmicos
  • Profissionais
  • Pessoais

Cada projeto deve apresentar:

  • Nome.
  • Breve descrição.
  • Contribuição pessoal. (* item mais importante)
  • Tecnologias utilizadas.
  • Link para o repositório.
  • Semestre em que foi feito.
Projetos Integradores (ABP) devem indicar claramente o semestre em que foram desenvolvidos. Exemplo: 1DSM – 1º Sem. 2026

Informações Complementares

Podem ser incluídos:

  • Formação acadêmica.
  • Cursos e certificações.
  • Conhecimentos técnicos.
  • Idiomas.
  • Hobbies e interesses.

Contatos

Informar apenas redes sociais profissionais como:

  • GitHub
  • LinkedIn
Não informar e-mail, telefone ou outros dados pessoais.

4. Vídeo de Apresentação

O portfólio é avaliado nos semestres pares:

  • 2DSM
  • 4DSM
  • 6DSM

Em cada avaliação, o aluno deverá produzir um vídeo de aproximadamente 1 minuto, no formato de pitch profissional.

O vídeo deve apresentar:

  • Quem é o aluno.
  • Áreas de interesse.
  • Contribuições individuais nas duas ABPs mais recentes.
  • Principais tecnologias utilizadas.

Os projetos abordados em cada avaliação são:

Avaliação Projetos
2DSM 1DSM e 2DSM
4DSM 3DSM e 4DSM
6DSM 5DSM e 6DSM

5. Processo de Avaliação

A avaliação do portfólio ocorre apenas nos semestres pares.

Caso o aluno esteja cursando um semestre ímpar, mas matriculado em alguma disciplina de semestre par, será considerada a última nota obtida na avaliação regular do portfólio.

Recomenda-se atualização contínua do portfólio ao longo do curso.

6. Tecnologias Permitidas

Permitidas

  • HTML5
  • CSS3
  • JavaScript

Não Permitidas

  • Frameworks com build ou empacotamento.
    • GitHub Actions (/workflows) está bloqueado.
  • Bibliotecas externas hospedadas por terceiros.
  • Aplicações com backend.
  • APIs próprias para armazenamento ou recuperação de dados.
Todo o conteúdo deve funcionar exclusivamente a partir dos arquivos armazenados no próprio repositório.
